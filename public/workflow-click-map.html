2) Cin7 Buttons and What Each Click Does

ButtonClick ActionNetwork CallUI Result
LC Freight Opens Freight Costing popup and auto-detects SKU(s), quantity, and shipping address from Cin7. Calls freight API endpoints from userscript. Shows freight price, ship-from line, product metrics, and Cin7 stock summary.
Refresh freight with these quantities Re-reads edited quantity rows in popup and re-runs quote. POST /get-freight, POST /api/product-metrics Updates freight total and status lines (X add to cart + Y PRE-SALE).
Site Visit Opens editable Site Visit popup prefilled from Cin7 fields. No call on open (local scrape only). User can edit fields before save.
Save Site Visit Card Validates form then posts site visit payload. POST https://living-culture-workflow.vercel.app/api/site-visits Saves card, auto-opens planner focused to branch/date/order card.
Quote Memo Info Builds quote memo text block with stock/pre-sale messaging and terms. Script-local logic + optional lookup Outputs memo content for quote notes/messages.
Install Fees Opens install fee helper panel for fee calculations and injection. Depends on helper script route/logic Creates/upgrades install fee rows in quote lines.

7) Fast Troubleshooting by Symptom

SymptomMost Likely CauseCheck First
Button missing in Cin7 Script disabled, wrong @match, stale script version Tampermonkey dashboard -> enabled + latest version + reload Cin7
Shows wrong stock status Old freight script cache or missing Cin7 stock in response Confirm script version; inspect network for /api/product-metrics payload
Site Visit saves but planner does not focus card Old site-visit script or old workflow deployment Check URL params appended after save; verify workflow deploy version
Cin7 non-JSON / empty response Wrong Cin7 base URL or credentials rejected Vercel env vars: CIN7_CORE_API_BASE_URL, CIN7_CORE_ACCOUNT_ID, CIN7_CORE_APPLICATION_KEY
Staff can’t get updates Script installed from old branch/pinned commit URL Ensure install/update URL uses /main/userscripts/...
